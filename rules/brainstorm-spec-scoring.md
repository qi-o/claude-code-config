# Brainstorm Spec Scoring

## 适用范围

- brainstorming skill 的 Step 7（方案评估）
- plan 模式的初始阶段（需求理解）
- 不替代 deep-interview 的完整模糊度评分

## 轻量模糊度评分

在 brainstorming Step 7 评估方案时，额外量化 4 个维度：

| 维度 | 权重 | 评分标准 |
|------|------|---------|
| 功能明确性 | 0.30 | 输入/输出/边界是否清晰定义 |
| 技术可行性 | 0.25 | 依赖是否确定、实现路径是否具体 |
| 非功能需求 | 0.25 | 性能/安全/可维护性要求是否明确 |
| 集成复杂度 | 0.20 | 与现有系统的接口是否确定 |

每个维度 0-1 分（0=完全模糊，1=完全明确）。

**综合评分** = Σ(维度分 × 权重)

**门控阈值**：
- ≥ 0.75 → 方案清晰，可进入 plan
- 0.50-0.74 → 标记薄弱维度，plan 中重点澄清
- < 0.50 → 回到 brainstorming 继续细化

## 与 deep-interview 的关系

| 场景 | 使用哪个 |
|------|---------|
| 快速方案比较 | 本规则（轻量） |
| 复杂需求分析 | deep-interview（完整） |
| plan 模式初始化 | 本规则（轻量） |
