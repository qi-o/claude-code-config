# Claude Command: Polish

在交付前对代码进行**质量优化**。侧重于人工审查和创意改进，不可自动化。

## 触发方式

```
/polish
```

## 执行步骤

### 1. 代码质量审查
- 检查函数是否过长（> 50 行）
- 检查嵌套深度（> 3 层）
- 检查重复代码（> 3 次）

### 2. 语义优化
- 变量/函数命名是否清晰表达意图
- 是否有 Magic numbers
- 注释是否准确、有意义

### 3. 结构优化
- 文件是否 < 300 行
- 模块职责是否清晰
- 依赖关系是否合理

### 4. 错误处理审查
- 关键操作是否有错误处理
- 错误信息是否泄露敏感数据

### 5. 业务逻辑审查
- 边界条件是否处理
- 异常流程是否覆盖
- 是否有 TODO 待完成

## 输出

完成后报告所有质量优化改动。
